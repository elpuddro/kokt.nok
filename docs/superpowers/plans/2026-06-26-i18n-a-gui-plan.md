# Tospråklig app — Sub-prosjekt A: GUI-lag Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Legg til `i18n.ts` med `t(key, lang)`, reaktiv `lang`-state i `+page.svelte` og språkvalg i Innstillinger — alle hardkodede norske UI-strenger erstattes med `t("key", lang)`.

**Architecture:** Ny `src/lib/i18n.ts` eksporterer `t(key, lang)`, `detectLang()` og `Lang`-typen. `+page.svelte` får én `lang: Lang = $state(...)` som lastes fra `innstillinger.json` (felt `spraak`) i `onMount`. Alle hardkodede UI-strenger erstattes med `t()`-kall. Språkbytte i Innstillinger er øyeblikkelig uten reload.

**Tech Stack:** SvelteKit 5 (runes), TypeScript, `@tauri-apps/plugin-store`, Vitest-lignende `.mjs` node-tester.

## Global Constraints

- Svelte 5 runes: `$state()`, `$derived()` — ALDRI `writable()`, `derived()` fra svelte/store
- Import-stier i `.svelte`: ALDRI `.ts`-suffix (skriv `$lib/i18n`, ikke `$lib/i18n.ts`)
- Tester: `.mjs`-filer, kjøres med `node src/lib/<filnavn>.test.mjs`
- CSS-variabler: `--bg-warm`, `--border`, `--accent`, `--text`, `--card` (ALDRI `--farge-*`)
- `innstillinger.json` Store: nytt felt `spraak: "nb" | "en" | "auto"` — default `"auto"` om feltet mangler
- Fengsel-flag: `erFengsel = import.meta.env.VITE_UTGAVE === 'fengsel'` — del-knapp allerede skjult bak dette
- INGEN Claude co-author trailer i commits
- Alkohol ekskludert fra alle forslag (allerede i Rust — ikke berøres)
- `detectLang()` bruker `navigator.language`: starter med `"nb"` eller `"nn"` → `"nb"`, alt annet → `"en"`
- Fallback-kjede i `t()`: `strings[lang][key]` → `strings["nb"][key]` → `key` selv (aldri tom streng)

# Bedre søk og sortering — Implementeringsplan (#5)

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Søkefeltet tolker mellomromsseparerte ord som AND-kombinasjon (hvert ord må matche navn eller ingrediens), og brukeren kan sortere oppskriftslisten på navn (A–Å/Å–A) og tilberedningstid (kortest/lengst).

**Architecture:** `hent_oppskrifter` i `lib.rs` utvides med AND-splitting av `sok` og ny `sorter`-parameter; tidssortering skjer i Rust-minnet via `tid_til_min()`. Frontend får en `<select>`-dropdown i `#main-header` og persisterer valget i Tauri Store (`sorter.json`).

**Tech Stack:** Tauri 2 (Rust, rusqlite), Svelte 5 runes (`$state`), TypeScript, `@tauri-apps/plugin-store`, `invoke` fra `@tauri-apps/api/core`.

## Global Constraints

- Svelte 5 runes kun — ingen `$:`, ingen `writable()`
- `invoke` fra `@tauri-apps/api/core` — ikke `@tauri-apps/api`
- Tauri Store: `load("sorter.json")`, `store.get/set/save()`
- Rust: ingen nye Cargo-dependencies — ingen regex-crate
- Ingen DB-endringer, ingen migrasjoner
- Maks 5 søkeord tolkes (ord utover 5 ignoreres stille)
- NULL-verdier i `tid` sorteres alltid sist (uansett retning)
- Commit-meldinger uten "Co-Authored-By"-trailer
- CSS: bruk `--card`, `--card-hover`, `--text-muted`, `--border`, `--surface`, `--bg` (ikke --card-bg, --hover-bg)

  ```rust
  fn tid_til_min(s: &str) -> Option<i64> {
      let s = s.trim().to_lowercase();
      if s.contains("time") && s.contains("min") {
          let t: i64 = s.split("time").next()?.trim().parse().ok()?;
          let m: i64 = s.split("time").nth(1)?.replace("min", "").trim().parse().ok()?;
          return Some(t * 60 + m);
      }
      if s.ends_with("time") {
          return s.replace("time", "").trim().parse::<i64>().ok().map(|t| t * 60);
      }
      if s.ends_with("min") {
          return s.replace("min", "").trim().parse::<i64>().ok();
      }
      None
  }
  ```

  ```rust
  fn tid_til_min(s: &str) -> Option<i64> {
      let s = s.trim().to_lowercase();
      // "X time(r) Y min"
      if (s.contains("timer") || s.contains("time")) && s.contains("min") {
          let del = if s.contains("timer") { "timer" } else { "time" };
          let t: i64 = s.split(del).next()?.trim().parse().ok()?;
          let m: i64 = s.split(del).nth(1)?.replace("min", "").trim().parse().ok()?;
          return Some(t * 60 + m);
      }
      if s.ends_with("timer") {
          return s.replace("timer", "").trim().parse::<i64>().ok().map(|t| t * 60);
      }
      if s.ends_with("time") {
          return s.replace("time", "").trim().parse::<i64>().ok().map(|t| t * 60);
      }
      if s.ends_with("min") {
          return s.replace("min", "").trim().parse::<i64>().ok();
      }
      None
  }
  ```

  ```rust
      let oppskrifter = if sorter_str == "tid_asc" || sorter_str == "tid_desc" {
          // Tidssortering: hent alle filtrerte rader, sorter i Rust, paginer manuelt.
          let alle_sql = format!(
              "SELECT o.id, o.slug, o.navn, o.type, o.porsjoner, o.tid, o.bilde
               FROM   oppskrifter o {where_sql}"

          let mut rader: Vec<serde_json::Value> =
              query_json(&conn, &alle_sql, filter_refs.as_slice())?;

          rader.sort_by(|a, b| {
              let ta = a["tid"].as_str().and_then(tid_til_min);

              match (ta, tb) {

          total,
  ```
